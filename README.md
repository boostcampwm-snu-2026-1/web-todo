# Web-Todo 프로젝트 학습 내용

## 1. npm 개념
npm이란 Node Pakage Manager의 약자.  
자바스크립트 프로그램(모듈/패키지)을 쉽게 설치, 업데이트, 삭제할 수 있도록 도와주는 관리 도구

**프로젝트 관리의 핵심 3대장**

① package.json
 : 프로젝트의 메타정보(이름, 버전 등)와 프로젝트가 의존하고 있는(설치한) 모듈들의 정보를 JSON 형태로 모아놓은 파일

② node_modules
 : 실제로 패키지들이 설치되는 디렉토리  
 package.json에 적힌 패키지뿐만 아니라, 그 패키지들이 작동하기 위해 추가로 의존하는 모든 하위 모듈들까지 전부 포함

```
의존성 트리(Dependency Tree) // package.json에 등록된 모듈과, 그 모듈들이 의존하는 모든 하위 모듈들의 관계를 쫙 펼쳐놓은 족보

 node_modules/
├── 바퀴 패키지             // 내가 package.json에 적어서 직접 설치한 모듈
│   ├── 고무 패키지         // 바퀴가 작동하기 위해 알아서 따라온 하위 모듈 1
│   │   └── 천연고무 원액   // 고무를 만들기 위해 또 따라온 최하위 모듈
│   └── 휠 패키지           // 바퀴가 작동하기 위해 알아서 따라온 하위 모듈 2
└── ... (수많은 패키지들)
```
따라서 용량이 매우 큼. Git 커밋 시에는 제외해야 함 (어차피 package.json만 있으면 언제든 다시 설치 가능하기 때문)

npm install express 실행시 전세계 개발자들이 만들어둔 Express 코드 꾸러미를 내 프로젝트의 node_modules 폴더 안에 다운로드 함. 
이것이 패키지를 '설치'한다는 의미

③ package-lock.json
 : 거대한 의존성 트리 구조가 생성된 딱 그 시점의 상태를 사진 찍듯이 기록해 두는 파일. 패키지 버전 업데이트로 인한 호환성 문제를 해결하기 위해 호환가능한 패키지 버전만 설치하도록 잠구는 역할

## 2. 로컬 서버와 클라우드 DB

로컬(Local)의 의미: 오직 내 컴퓨터 안에서만 접속 가능하다는 것. 아직 이 서버를 AWS 같은 실제 인터넷 공간에 올리지(배포하지) 않았기 때문에, 외부 사람은 접속할 수 없고 내 컴퓨터의 크롬 브라우저에서만 localhost:3000으로 접속할 수 있음

터미널에 친 npm run server라는 명령어는 Node.js라는 프로그램에게 "내 index.js 파일을 실행해 줘!"라고 명령한 것
index.js 안에 있는 app.listen(3000)이라는 코드가 실행되면서 컴퓨터는 "지금부터 나는 3000번 포트(문)를 열어두고, 누군가 접속하기를 기다리는 서버다!" 하고 대기 상태에 들어가게 됨

프론트엔드 화면(Vite)과 백엔드 서버(Express)는 둘 다 내 컴퓨터(로컬) 안에서 돌아가고 있음

만약 데이터베이스(DB)마저 내 컴퓨터에 직접 설치해서 쓴다면, 내 컴퓨터가 꺼지는 순간 사이트도 죽고 데이터도 날아가게 됨. 그래서 데이터를 안전하게 보관하기 위해 클라우드 DB(MongoDB Atlas) 를 사용함. 직접 장비를 구매하고 내 컴퓨터에 구축하는 방식에서 벗어나, 외부의 거대한 서버 환경을 대여해서 사용하는 것.

💡 식당에 비유한 전체 흐름
프론트엔드 (Vite / 로컬): 식당의 손님. "할 일 목록 추가해 줘!" 하고 주문을 넣음.
백엔드 (Express / 로컬): 식당의 종업원. 손님의 주문을 받아서 주방으로 전달. 
클라우드 DB (MongoDB / 인터넷): 인터넷 어딘가에 있는 거대한 식자재 창고. 종업원(백엔드)이 인터넷망을 타고 이 창고에 접속해 데이터를 저장하거나 꺼내서 손님(프론트엔드)에게 가져다 줌.
